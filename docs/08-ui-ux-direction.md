# UI / UX 方向文档

## 设计命题

GitMob 需要的是一种会让人愿意打开的代码审查体验，而不是一个信息压缩得很满的移动 GitHub 客户端。

新的命题是：

> 把代码审查做成一套有 Tinder 感的、青春活力的、全屏内容驱动的交互体验。

## 核心判断

### 我们要像 Tinder 的地方

- 卡片是内容主角
- 当前对象始终处于视觉中心
- 动作反馈清楚且带情绪
- 下一步行动很明确
- 用户有“继续刷下去”的节奏感

### 我们不能像 Tinder 的地方

- 不能为了爽感牺牲信息精度
- 不能让颜色和动效压过代码本身
- 不能让评论和 review 提交流程变得轻浮

## 空间策略

### 总原则

- 核心流不用 header
- 核心流不用 footer
- 顶部信息变成轻量 pill
- 底部操作变成 action dock
- 复杂信息通过 sheet 进入

### 屏幕空间分配

#### Feed

- 70%：主 PR 卡
- 15%：辅助预览条
- 15%：utility 和浮动入口

#### Review Deck

- 65%：当前 segment card
- 20%：代码 peek / 次级信息引导
- 15%：action dock 和 handle

#### Code Peek Sheet

- 80%：diff 内容
- 20%：meta + 评论入口

## 视觉语言

### 风格名

`Sunset Pop x Code Glass`

### 语气

- 年轻
- 有生命力
- 有弹性
- 不装成熟
- 不像企业管理台

### 元素关键词

- 柔和渐变
- 珊瑚橙高光
- 青绿色点亮
- 胶囊按钮
- 玻璃感浮层
- 大弧角
- 微颗粒背景
- 轻反光卡片边缘

## 组件情绪

### Accept

感觉：

- 温暖
- 被点亮
- 正反馈

表现：

- 暖橙或珊瑚发光
- 向右的轻位移
- 柔和震动

### Concern

感觉：

- 拉住
- 提醒
- 紧张但不恐怖

表现：

- 番茄红
- 轻微压缩
- 稍强触感

### Comment

感觉：

- 被抬起
- 进入更深一层

表现：

- 青绿色抬升
- sheet 弹出
- 焦点落到输入区

## 文字系统

### 字体层级

- Hero / Card Title: `Bricolage Grotesque`
- UI Body: `Instrument Sans`
- Code: `JetBrains Mono`

### 文字规则

- 卡片标题短而有力
- summary 不超过 3-4 行
- 元信息优先用 badge / pill 表达
- 不在主卡里堆大量字段标签

## 导航语言

### 顶部

只允许出现：

- back pill
- progress pill
- repo pill
- avatar orb

### 底部

只允许出现：

- action dock
- handle
- 临时 CTA

### 不允许出现

- 常驻 tab bar
- 大块标题栏
- 页眉里塞描述、状态、筛选、搜索全部信息

## 页面蓝图

### 1. PR Feed

视觉特征：

- 主卡居中偏上
- 下一张卡片部分露出
- 背景有轻微暖色氛围
- 顶部只是一排小 utility pills

### 2. Review Deck

视觉特征：

- 当前卡片占满注意力
- 下一张卡片只露出边缘或阴影
- 底部 action dock 像悬浮控制器，不像 footer
- code peek 用 sheet 从底部出现

### 3. Outcome Sheet

视觉特征：

- 有完成一轮选择后的收束感
- 数字和结果一眼可读
- CTA 明确但不过度威胁

## 可用性底线

即使视觉方向更激进，也必须满足：

- 代码区域高对比
- 评论锚点稳定
- 关键信息不只靠颜色区分
- 所有主要动作可以按钮触发，不强依赖手势
- 大 PR 时性能不明显下降

## 实施建议

### 第一阶段

先做结构，不先做装饰：

- 去掉 header/footer
- 建立 shell
- 建立 feed / review / outcome 三屏结构

### 第二阶段

再做表现层：

- 色板
- 字体
- 高光和渐变
- 动效节奏
- 触觉反馈

### 第三阶段

最后做精修：

- 卡片层叠比例
- 胶囊尺寸
- safe area 贴边距离
- 字体节奏
- 按钮反馈强弱
